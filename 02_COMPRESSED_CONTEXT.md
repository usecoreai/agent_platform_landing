# CoreAI Platform — сжатый контекст обсуждения

## 1. Ситуация

Команда CoreAI делает enterprise-ready агентскую платформу / агентский хаб с коннекторами к корпоративным системам, хорошим менеджментом контекста, выбором разных LLM-провайдеров, встроенными скиллами, role/admin control, логированием и перспективой A2A + pipeline builder.

Текущий фокус: сделать сайт `platform.core-ai.ru`, который будет конкурировать по категории с Ladcraft, MWS AI Agents Platform, Just AI Agent Platform и native AI-слоями в CRM/ERP. При этом нужно не копировать конкурентов, а занять свою category frame.

## 2. Главная позиция

CoreAI Platform — не еще одна CRM, ERP, BI или база знаний. Это агентский слой / Second Brain компании поверх уже внедренных корпоративных систем.

Главная формула:

> Хранить — задача CRM/ERP.  
> Думать, связывать и действовать — задача CoreAI.

Расширенная формула:

> Не внедряйте ещё одну систему учёта. Подключите CoreAI как агентский слой поверх 1С, CRM, SQL, документов и внутренних сервисов — с пайплайнами, скиллами, A2A-агентами, выбором моделей и enterprise-контролем.

Категорийное позиционирование:

> CoreAI Platform — Second Brain компании поверх 1С, CRM, SQL, документов и внутренних систем.

Важно: Second Brain не должен звучать как обычная база знаний. Это **agentic second brain**: память + reasoning + agents + pipelines + actions + enterprise control.

## 3. Почему это сильнее, чем “AI-чат”

Обычный AI-чат отвечает на вопрос. CoreAI должен показывать более высокий уровень:

1. Сотрудник или руководитель ставит цель обычным языком.
2. Система находит контекст в 1С, SQL, CRM, документах, задачах и внутренних сервисах.
3. Координатор строит план.
4. Специализированные агенты выполняют части задачи.
5. Pipeline вызывает tools и коннекторы.
6. Ответ сопровождается источниками, расчетами, графиками и trace.
7. Действие может быть подготовлено под approval.
8. Все сохраняется в audit log.

Ключевая формула из презентации DeepDesk, которую нужно сохранить: **Ask · Verify · Act · in a single window**.

## 4. Текущий MVP

Что реально есть / должно быть к ближайшему MVP:

- базовый агентский чат;
- роли пользователя и администратора;
- логирование;
- интеграция с 1С read-only через OData;
- SQL-коннектор;
- возможность добавлять коннекторы архитектурно;
- построение графиков;
- дашборд, куда можно добавлять графики из получаемых ответов.

Что ожидается в alpha / ближайшие 2–3 недели:

- Skill Hub;
- basic Pipeline Builder;
- multi-agent orchestration v0;
- A2A-ready architecture;
- provider selector;
- улучшенный context panel;
- trace/audit timeline;
- CRM connector layer / demo CRM на синтетических данных;
- Telegram notifications;
- CSV/XLSX upload;
- action preview UI.

Важно для сайта: не использовать слабые бейджи MVP/Alpha в первом экране. Верхнеуровнево заявлять как **Early Access / доступно для пилотов / первый релиз платформы**, а в FAQ честно объяснять, что состав коннекторов и deployment зависят от сценария пилота.

## 5. 1С-позиция

Партнерского статуса с 1С пока нет. Поэтому нельзя писать “официальный партнер 1С” или использовать логотипы без прав.

Можно писать:

- “коннекторы и сценарии для работы с системами 1С”;
- “глубокая адаптация под 1С-контур”;
- “read-only подключение к 1С через OData в раннем релизе”;
- “переход к controlled actions: preview → approval → execution → audit”.

Очень важно: запись в 1С нельзя подавать как “агент меняет любые данные”. Правильная рамка:

> Сначала read-only. Затем controlled actions через preview, human approval и audit.

## 6. Конкуренты и что брать из их упаковки

### Ladcraft
Прямой ориентир по категории. Они продают автономных AI-агентов, skills, память, команду агентов, интеграции, 1С и внедрение через ROI-пилоты. Их сильная сторона — enterprise packaging и понятные бизнес-роли.

CoreAI должен конкурировать не в “у нас тоже автономные агенты”, а в более системной рамке:

> CoreAI — не просто AI-сотрудник. Это Second Brain и агентский слой поверх всех систем компании.

### Just AI Agent Platform
Сильный enterprise/no-code/pro-code конкурент: agent platform, on-prem, RAG, визуальные редакторы, governance.

### MWS AI Agents Platform
Сильный heavy enterprise: lifecycle, AgentOps, LLMOps/MLOps, observability, role model, security, ready assistants.

### amoCRM / Bitrix24 AI / CRM AI-визжеты
Закрывают узкие sales-use-cases внутри одной CRM. Это не прямая платформа, но они учат рынок формуле “AI-сотрудник внутри привычной системы”. CoreAI нужно быть выше:

> Не AI-виджет внутри одной CRM, а cross-system agentic layer поверх CRM + 1С + SQL + документов.

## 7. Кто первый ICP сайта

Сайт должен бить в несколько аудиторий, но в hero прежде всего в предпринимателя / собственника / CEO / COO:

- “не внедряйте еще одну систему учета”;
- “выжмите максимум из уже внедренных систем”;
- “получите Second Brain компании”;
- “руководитель ставит цель и получает проверенный результат”.

Ниже закрываются:

- CIO/CISO: control, audit, on-prem, RBAC, provider selection.
- 1С/CRM-интеграторы: “усильте вашу экспертизу AI-платформой”.
- Консалтинги: “запускайте AI-пилоты и ROI-blueprints без сборки платформы с нуля”.

## 8. Главный демо-сценарий

Нужно уйти от “чат построил график”. Главный demo-flow:

> Одна бизнес-цель → несколько систем → несколько агентов → pipeline → дашборд → audit.

Пример запроса:

> “Сравни план-факт из 1С и SQL, проверь сделки без next step в CRM, найди причины отклонений и собери дашборд для руководителя.”

Что должно быть видно:

1. Coordinator Agent понял цель.
2. Data Agent пошёл в 1С/SQL.
3. Sales Agent проверил CRM.
4. Finance Agent нашёл отклонения.
5. Report Agent собрал выводы.
6. Pipeline сохранил график на dashboard.
7. Audit log зафиксировал источники, tool calls и шаги.

## 10. Визуальный стиль

Ориентир:

- CoreAI/DeepDesk стиль: белый/графит/черный + фирменный оранжевый;
- много воздуха;
- интерфейсные mockups вместо абстрактных AI-сфер;
- ощущение enterprise infrastructure, не “AI magic”; 
- можно использовать мотивы “second brain / mission control / control plane / agentic workflow”.

## 11. Важная осторожность

Сайт должен быть амбициозным. Но нельзя выглядеть как “лендинг обогнал продукт на год”. Поэтому:

- в hero — категория и возможности Early Access;
- в FAQ — честное уточнение, что набор интеграций и режим deployment зависят от пилота;
- critical enterprise features можно маркировать “доступно в enterprise-пилотах / roadmap / designed for”.

Главное — сделать один законченный спектакль платформы в демо:

> цель → агенты → источники → пайплайн → график → дашборд → лог → следующий шаг.
